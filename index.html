2Ticket Triage (Must Complete Within 1 Hour)

Step-by-Step Triage Process

1. Assignment

Views → Unassigned Tickets → Select Ticket(s) → Edit → Assignee → CX → [Name] → Submit

2. Customer Type (if not auto-assigned)

• Operator - Property management companies using Revyse

• Vendor - Service providers in marketplace

3. Ticket Type

• Question - General how-to or clarification

• Incident - System outage, bug, or broken functionality

• Task - Action required (uploads, data cleanup, account changes)

• Problem - Recurring issue affecting multiple customers

4. Priority Assessment Matrix

Impact Urgency Priority Examples
High High 🔴 Urgent System down, can't access platform, blocking onboarding
High Low 🟠 High Feature broken but workaround exists
Low High 🟡 Normal How-to question needed quickly
Low Low 🟢 Low General questions, feature requests

5. Product Area Assignment

• Contract Management (Zipline) - Renewals, uploads, line items, contract lifecycle

• Vendor Credentialing - Insurance docs, compliance, risk tiers, certifications

• Discovery Marketplace - Vendor listings, subscriptions, reviews, search

• Contract Execution - Templates, countersigners, dynamic variables, e-signatures

• Property Docs - Inspection docs, licenses, NFPA, property-level uploads

• Billing - RVI invoicing and payment issues (Operators only)

• Talk to Sales - Demo requests, expansion conversations

• Other/Escalation - Technical bugs, feature requests, unusual scenarios

3Ticket Status Workflow

Status Definitions & Actions

New

• System default for incoming tickets

• Action Required: Triage within 1 hour, assign within SLA first-response time

• SLA Status: Clock running

Open

• Agent ownership assigned, queued for work

• Action Required: Begin work or provide initial response within SLA

• SLA Status: Clock running

In Progress

• Agent actively working on request

• Action Required: Update customer within SLA timeframe

• SLA Status: Clock running until customer response sent

Pending

• Waiting on customer response or external dependency

• Action Required: Set follow-up reminders, document what you're waiting for

• SLA Status: Clock paused

Solved

• Resolution provided to customer

• Auto-behavior: Reopens to "Open" if customer replies within 4 days

• SLA Status: Clock stopped

4Communication Standards

Humility & Authenticity: No fluff—just authenticity. We show up as our genuine selves, balancing professionalism with being human.

Our Voice & Tone

Think: Casual coffee chat with a multifamily professional who needs real help, not corporate-speak

Do

• Keep it conversational, not corporate
• Be friendly without being overly formal
• Stay optimistic and solution-focused
• Include KB links naturally: "Here's the guide that covers this →"
• Set clear expectations like texting a trusted colleague

Don't

• Overuse exclamation points (we're helpful, not cheerleaders)
• Use jargon that makes customers Google on the side
• Make vague promises like "we'll look into it"
• Write legal briefs when a simple explanation works
• Hide behind corporate walls when things go wrong
